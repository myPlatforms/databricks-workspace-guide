4 Cross-Account IAM Role 생성

IAM → Roles → Create role → Custom Trust Policy

{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": {
      "AWS": "arn:aws:iam::414351767826:root"
    },
    "Action": "sts:AssumeRole",
    "Condition": {
      "StringEquals": {
        "sts:ExternalId": "<YOUR_DATABRICKS_ACCOUNT_ID>"
      }
    }
  }]
}

Backend Private Link

Compute Plane → Control Plane 전용 연결

AWS Backend Private Link 아키텍처

목적: Compute Plane에서 Control Plane으로의 통신을 AWS 내부 네트워크(Private Link)를 통해 수행

Option 1 vs Option 2

Option 1: NAT Gateway 포함

  • Regional NAT Gateway + Internet Gateway 배포
  • STS, Kinesis는 공용 인터넷 경유
  • 구성이 비교적 단순
  • NAT Gateway 비용 발생

Option 2: 완전 Private (NAT 없음)

  • NAT Gateway 없음 → 공용 인터넷 미사용
  • STS, Kinesis용 VPC Endpoint 추가 필요
  • 더 높은 보안 수준
  • 추가 Endpoint 비용 발생

1 Endpoints 서브넷 구성

Customer VPC 내에 Databricks 서비스 Endpoint용 전용 서브넷 생성

  • databricks-endpoints-subnet: Databricks 서비스 (REST API, SCC) Endpoint용
  • aws-endpoints-subnet (Option 2만): STS, Kinesis Endpoint용

VPC → Subnets → 서브넷 생성 화면

3 REST API VPC Endpoint 생성

VPC Dashboard → Endpoints → Create endpoint

  1. Service type: services that use NLBs and GWLBs
  2. Service name: Regional Workspace (REST API) 서비스명 입력
    예: com.amazonaws.vpce.us-west-2.vpce-svc-xxxxxxxxx
  3. VPC: Customer VPC 선택
  4. Subnet: databricks-endpoints-subnet 선택
  5. Security group: databricks-backend-endpoints-securitygroup
  6. Enable private DNS name 활성화
AWS VPC Endpoint ID를 반드시 기록하세요! (Databricks Account Console 등록 시 필요)

4 SCC Relay VPC Endpoint 생성

동일 절차로 Secure Cluster Connectivity (SCC) Relay Endpoint 생성

  • 이름에 scc 포함 권장 (예: databricks-us-west-2-scc-vpce)
  • 같은 서브넷 및 Security Group 사용

참고: 경우에 따라 Customer VPC와 Transit VPC를 하나의 VPC로 통합할 수 있으며, 이 경우 동일한 REST API Endpoint를 Backend/Frontend 모두에 재사용합니다.

1 Frontend VPC Endpoint 생성

Transit VPC에서 VPC Endpoint 생성

  1. Service type: services that use NLBs and GWLBs
  2. Service name: Regional Workspace (REST API) 서비스명

  3. VPC: Transit VPC 선택
  4. Subnet: Transit VPC의 Private Subnet 선택
  5. 이름 예: databricks-us-west-2-workspace-fe-vpce
AWS VPC Endpoint ID를 기록하세요 (Frontend-REST-API-endpoint)

2 Account Console에 Frontend Endpoint 등록

Account Console → Security → Connectivity → VPC Endpoints

  1. Register VPC endpoint 클릭
  2. 설명적인 이름 입력 (Frontend임을 구분할 수 있도록)
  3. Region 선택
  4. Frontend-REST-API-endpoint AWS VPC Endpoint ID 입력
  5. 등록 완료

3 Private Access Settings 생성

Account Console → Security → Connectivity → Private access settings

  • Private Access Settings 객체는 워크스페이스의 Frontend Private Link 연결을 정의
  • 설정 항목:
    • Public Access: Disabled (Private Link만 허용) 또는 Enabled
    • Private Access Level: Account / Any
  • 워크스페이스에 이 설정을 적용하여 Frontend PL 활성화

3 NSG (Network Security Group) 설정

Azure Databricks가 자동으로 필수 규칙을 추가하지만, 커스텀 VNet 사용 시 확인 필요

필수 NSG 규칙:
  • Databricks Control Plane → Data Plane: TCP/443, TCP/8443-8451
  • Data Plane → Control Plane: TCP/443
  • Data Plane → Azure Storage: TCP/443
  • Data Plane → Azure SQL (Metastore): TCP/3306
  • Worker ↔ Worker: Any (내부 통신)

Azure Portal → Private Link Center → Private endpoints → Create

자동 DNS: Private Endpoint 생성 시 "Integrate with private DNS zone" 옵션을 활성화하면 DNS Zone 생성 및 레코드 등록이 자동으로 처리됩니다.

3 Workspace 배포 설정

워크스페이스 배포 시 또는 배포 후 네트워크 설정에서:

Azure Frontend Private Link 아키텍처

1 Transit VNet 구성

3 Private DNS Zone 연결

  • 기존 privatelink.azuredatabricks.net DNS Zone에 Transit VNet 연결 추가
  • 또는 새로운 DNS Zone 생성 후 Transit VNet에 연결
  • On-prem DNS에서 Azure Private DNS로 조건부 전달(Conditional Forwarding) 설정
On-prem DNS: On-prem 사용자가 Private Endpoint IP로 해석되려면 DNS Forwarder (Azure DNS Private Resolver 등)가 필요합니다.

4 공용 네트워크 접근 비활성화

Azure Portal → Databricks Workspace → Networking

  • Allow public network access: No로 설정
  • 이후 모든 접속은 Private Endpoint를 통해서만 가능
  • Required NSG rules: NoAzureDatabricksRules 선택 가능

5 접속 확인

검증 단계:
  1. Transit VNet/On-prem에서 nslookup {workspace-url}
  2. Private IP (10.x.x.x)로 해석되는지 확인
  3. 브라우저에서 워크스페이스 URL 접속
  4. 외부 네트워크에서 접속 불가 확인 (공용 접근 비활성화 시)
# DNS 해석 확인 예시
$ nslookup adb-1234567890123456.16.azuredatabricks.net
Address: 10.0.1.4  # Private IP로 해석됨
